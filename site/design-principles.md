# 01MVP 极简设计原则：反信息堆叠指南

本指南基于 01MVP 技能站（skills.01mvp.com）的重构经验总结。旨在帮助未来的页面设计避免“信息堆叠”的反模式，保持硬核、克制、直击痛点的极简风格。

## 1. 核心理念 (Core Philosophy)

**极简不是因为缺少内容，而是因为去除了干扰。**
> “Perfection is achieved not when there is nothing more to add, but when there is nothing left to take away.”

01MVP 的页面应该像终端命令行一样：没有废话，意图明确，输入即执行。

## 2. 常见反模式与解决方案 (Anti-patterns & Solutions)

### 反模式一：自嗨型数据堆叠 (Vanity Metrics Stacking)
**现象：** 页面顶部放一堆对用户没有实质意义的数字（例如：`4 Core Skills`, `1 System`, `0->1 Execution`）。
**问题：** 用户并不关心你有几个组件，他们只关心这东西能帮他们解决什么问题。
**解决方案：** **直接删除。** 不要用数字填充版面，如果数字不能直接转化为用户的业务价值（如“为你节省 4 小时”），就不要放。

### 反模式二：分屏左右互搏 (Split-screen Clutter)
**现象：** 左边讲大词（BUILD WITH SKILLS），右边强塞一个深色的操作面板（Quick Start）。
**问题：** 视线跳跃，主次不分。用户不知道该看左边还是看右边。
**解决方案：** **单列垂直居中布局。** 让用户的视线从上到下顺畅流动：主标题 -> 副标题 -> 操作按钮。把所有的注意力收敛到一条中轴线上。

### 反模式三：过度解释 (Over-explaining)
**现象：** 专门开辟一个“核心价值”章节，写 4 个格子来论证为什么要用这个工具（结构化工作流、设计资产共享...）。
**问题：** 真正的极简工具不需要长篇大论的辩护。用大段文字说服用户，反而显得不自信。
**解决方案：** **删减章节。** 将核心价值浓缩到首屏的一句话（如：“不再需要每次重新输入冗长的背景”）。让产品自己说话。

### 反模式四：网格强迫症 (Grid Obsession)
**现象：** 把所有特性或列表硬塞进 2 列或 3 列的网格（Grid）中，导致有些格子内容多被撑开，有些格子很空，整体像一堵墙。
**问题：** 缺乏呼吸感，且不利于后续扩展（加一个单数内容就会破坏排版）。
**解决方案：** **列表式（List）布局。** 对于功能说明，使用上下堆叠的单列 List（左图右文或左文右代码）。这种排版更像是一份优雅的文档，而不是推销广告。

### 反模式五：Logo 滥用 (Logo Overuse)
**现象：** 页面顶部放一个硕大的图形 Logo，底部又放一个。
**问题：** 抢夺视觉焦点。在已经有文字表明品牌（如“01MVP / Skills”）的情况下，重复图形显得啰嗦。
**解决方案：** **文字替代图形。** 01MVP 的风格是“文字即 UI”。使用简单的纯文字 Label（如 `<span class="eyebrow">01MVP / Skills</span>`）替代复杂的 SVG Logo，显得更加自信和硬核。

## 3. 排版与视觉基准 (Typography & Spacing)

- **呼吸感 (Whitespace)：** `margin` 和 `padding` 永远往大了给。将区块间距从 `24px` 放大到 `64px` 甚至 `100px`，让信息有足够的空间呼吸。不要让文字和文字打架。
- **色彩克制 (Color Discipline)：** 全站只用黑（`var(--black)`）、白（`var(--white)`）和浅灰（`var(--gray)`）。绝对不使用任何彩色的背景或渐变。
- **边框与阴影 (Borders & Shadows)：** 使用硬边框（`2px` 或 `3px solid black`）和硬阴影（无模糊半径的实色偏移）。绝对不要用 `border-radius`（零圆角）。
- **按钮层级 (Button Hierarchy)：** 一个视口（Viewport）内，只有一个最显眼的动作。例如：首屏唯一的重色输入框就是那条 `npx` 安装指令。

## 4. 终极检验法则 (The Ultimate Test)

在提交任何 UI 设计前，问自己：
**“如果我把这个元素删掉，用户还能不能理解这是什么、该怎么用？”**
如果答案是能，那么立刻删掉它。
