# CheatSheet 生成器

请你产出一份「[工具名称]」的新手友好型 Cheatsheet（10 分钟内读完）。目标是：让读者在最短时间内明白为什么要用它，并在 3 个步骤内跑通第一个可见结果。

## 写作总则（必须遵守）
- 说人话：避免堆砌术语，能用日常表达就不用专业名词；不要喊口号、不要鸡汤。
- 开门见山：先回答"为什么值得用 + 怎么立刻用起来"，再讲细节。
- 结果导向：所有命令必须可复制粘贴，并在行内注释标注作用与预期结果。
- 控制篇幅：全文不超过 1200 字；若内容较多，优先裁剪低频功能并放到"继续深入"。
- 可读性：使用 Markdown 标题结构，代码块配简要注释；默认使用 bash。

## 输出结构（严格按此顺序）

### ## 为什么用它
- 一句话价值主张：如果你想在【场景/目标】用最少操作做到【结果】，【工具名称】能在【Z 分钟】内帮你完成。
- 三个立刻能感知的收益：越具体越好，避免空话。
- 适合/不适合：各 1–2 条，帮助用户快速判断。

### ## 3 步极速上手
目标：让用户复制粘贴命令就能看到效果，首次示例要简单有效。

**步骤 1：安装**
```bash
[简单的安装命令]  # 优先选择最主流的安装方式
```

**步骤 2：创建最简示例**
```bash
[最简初始化命令]    # 创建最小可运行示例，避免复杂配置
[启动命令]          # 本地启动，确保能立即看到结果
```

**步骤 3：验证成功**
```bash
[验证命令或访问地址]  # 明确的成功判定标准
```

> 每一步都必须写清：命令作用、成功判断标准。首次示例应该是"Hello World"级别的简单。


**[功能分类2]**


包含 3-5 个有用但使用频率较低的命令：

**[功能分类]**


给出能跑通所需的最基础配置文件：


- 解决：

3. 刷新验证修改生效

记录遇到的问题和解决方案，建立实战经验。

## 写作风格提醒
- 新手第一，能立刻上手；把原理/细节延后到"继续深入"
- 命令必须可复制执行，避免抽象占位符
- 每个代码块前用一句话说明用途
- 不确定是否必要的内容，一律不写
